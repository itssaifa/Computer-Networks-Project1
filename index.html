Website Setup & Networking Report

Domain & Hosting

For hosting and domain setup, GitHub Pages was used. It provides a free and simple way to host static websites directly from a GitHub repository. By pushing the project files to a public repository and enabling GitHub Pages, the website was deployed with a free subdomain.

Website Setup

The website was built using a combination of HTML and CSS to ensure a clean and neat layout. No JavaScript or other frameworks were used, keeping the site lightweight and easy to maintain.

Networking Aspects

The site is publicly accessible online through GitHub Pages. GitHub handles a lot of the networking configuration, including setting up DNS records and generating an IP address for the hosted site. This demonstrates key networking principles such as:

  • DNS (Domain Name System): GitHub Pages assigns a unique subdomain under github.io, translating URLs to IP addresses.
  • IP Addresses: The hosted website is served from GitHub's IP range, making it reachable worldwide.
  • Protocols: The site is accessible via HTTPS, ensuring secure communication between users and the server.
